Meredith McNulty

Highlight #1 Version Control

Version control is a great way to record changes made to a file or set of files so that a user or multiple users can go back and look at past versions if necessary. This is a great tool for collaboration so that multiple people can work on a project at once without losing any important data. With version control, you can revert back to any previous version and also compare changes over time. If a file is lost, it can be retrieved through version control. The version control system can be local, remote, or a combination of the two.

Highlight #2 Git Basics

One way to think of Git is that it’s a method of organizing data into snapshots of a miniature filesystem. Every time a project or file is saved, a snapshot is taken and is stored as a reference to look back at when newer edits are made. Git uses local files and resources to run and operate. This makes the process much smoother and faster than having to reach out to a network or server. Change history is read directly out of a local database. Git is based on adding, not deleting. This prevents data from becoming lost or unrecoverable.

Highlight #3 Repositories

Getting a Git repository can be done two main ways: taking out an existing project or cloning from another server. The first method requires that the user to go into the project’s directory and then “$ git init” to create a new subdirectory named .git to contain all the necessary repository files. Cloning an existing repository can be done using the command “git clone”. This pulls every version for every file for the history of the project. This command doesn’t just get a working copy, it is able to receive nearly every piece of data that a server may have.

Highlight #4 Recording Repository Changes

Once you have the initial setup for Git, a user is able to begin working, making changes, and recording those changes through snapshots each time the project get to a point a user wants to make note of. Files are categorized into a tracked or untracked status. Tracked files are what the user records in a snapshot. These files are modifiable as well. Untracked files are any files in the working directory that are not in the previous snapshot and not in a staging area. A great method of checking the status of untracked, unmodified, modified, and staged files is using the command “git status”. This will show if you have any changes that are new and haven’t been tracked yet.

Highlight #5 Github Accounts & Features

Typically, Git repositories are hosted on Github, a web based version control system that has all the features of Git and more. It is the largest host for Git repositories and provides a central point for collaboration for its users. It’s possible to use Git without using GitHub, however GitHub is designed to make life easier on its user. It is account based, so a user creates a profile, and from there can clone public projects, fork projects, and push to forks as well. Now a user can add email address to use for commit messages and general information as well as set the security preference.
